# 2. Git

## 2.1 O que é o Git?

O **Git** é um sistema de controle de versão distribuído (DVCS - Distributed Version Control System) criado por **Linus Torvalds** em 2005. Ele permite que desenvolvedores rastreiem e gerenciem alterações em projetos de software de forma eficiente.

### Características Principais:

| Característica | Descrição |
|---------------|-----------|
| **Distribuído** | Cada desenvolvedor tem uma cópia completa do repositório |
| **Gratuito** | Software open-source sob licença GPL |
| **Rápido** | Operações locais são instantâneas |
| **Seguro** | Usa SHA-1 para garantir integridade dos dados |
| **Flexível** | Adapta-se a diversos fluxos de trabalho |

---

## 2.2 Como o Git Funciona?

O Git armazena o projeto em **snapshots** (commits). Cada commit representa o estado completo do projeto em um momento específico.

### Estrutura de Armazenamento:

```
Working Directory → Staging Area → Repositório Git
       ↓                  ↓              ↓
  Seus arquivos    Área de preparo   Commits salvos
```

### O Ciclo de Vida dos Arquivos:

```
┌─────────────┐    git add    ┌─────────────┐   git commit   ┌─────────────┐
│  Untracked  │ ────────────→ │   Staged    │ ─────────────→ │  Committed  │
│ (Não rast.) │               │ (Preparado) │                │  (Salvo)    │
└─────────────┘               └─────────────┘                └─────────────┘
       ↑                           ↑                                │
       │                    git reset                               │
       │                           │                                │
       └───────────────────────────┴────────────────────────────────┘
                          git checkout
```

### Estados dos Arquivos:

| Estado | Descrição | Comando para Mudar |
|--------|-----------|-------------------|
| **Untracked** | Arquivo novo, não monitorado | `git add` |
| **Tracked** | Arquivo já monitorado pelo Git | - |
| **Modified** | Arquivo alterado, não salvo | `git add` |
| **Staged** | Arquivo preparado para commit | `git commit` |
| **Committed** | Arquivo salvo no histórico | - |

## 2.3 Instalação do Git

### Windows

1. Baixe o instalador em: [https://git-scm.com/download/win](https://git-scm.com/download/win)
2. Execute o instalador e siga as instruções
3. Escolha as opções padrão (recomendado para iniciantes)

### macOS

```bash
# Usando Homebrew
brew install git

# Ou usando Xcode Command Line Tools
xcode-select --install
```

### Linux

```bash
# Debian/Ubuntu
sudo apt update
sudo apt install git

# Fedora/RHEL
sudo dnf install git

# Arch Linux
sudo pacman -S git
```

### Verificando a Instalação

```bash
git --version
# Saída: git version 2.43.0
```

## 2.4 Configurações Básicas do Git

Após instalar, configure suas informações pessoais:

### Configuração Global (todos os repositórios)

```bash
# Nome do usuário
git config --global user.name "Seu Nome Completo"

# E-mail (use o mesmo do GitHub)
git config --global user.email "seu.email@exemplo.com"

# Editor de texto padrão
git config --global core.editor "code --wait"  # VS Code
# ou
git config --global core.editor "nano"         # Nano
# ou
git config --global core.editor "vim"          # Vim

# Ver todas as configurações
git config --list
```

### Configuração Local (apenas um repositório)

```bash
# Entre na pasta do repositório
cd /caminho/do/repositorio

# Configurar e-mail diferente para este projeto
git config --local user.email "email.especifico@projeto.com"
```

### Níveis de Configuração:

| Nível | Comando | Escopo |
|-------|---------|--------|
| **System** | `--system` | Todos os usuários do sistema |
| **Global** | `--global` | Todos os repositórios do usuário |
| **Local** | `--local` | Apenas o repositório atual |

## 2.5 Comandos Básicos do Git

### Inicialização e Clone

```bash
# Iniciar novo repositório
git init

# Clonar repositório existente
git clone https://github.com/usuario/repositorio.git

# Clonar em pasta específica
git clone https://github.com/usuario/repositorio.git minha-pasta
```

### Verificando o Status

```bash
# Ver status dos arquivos
git status

# Ver status de forma resumida
git status -s
```

**Saída do `git status -s`:**
```
 M arquivo_modificado.txt    # Modificado, não adicionado
A  novo_arquivo.txt         # Adicionado ao staging
M  arquivo_staged.txt       # Modificado e no staging
?? arquivo_novo.txt         # Não rastreado (untracked)
```

### Adicionando Arquivos

```bash
# Adicionar arquivo específico
git add arquivo.txt

# Adicionar todos os arquivos
git add .

# Adicionar todos os arquivos .py
git add *.py

# Adicionar pasta inteira
git add pasta/
```

### Criando Commits

```bash
# Commit com mensagem curta
git commit -m "Mensagem do commit"

# Commit com mensagem detalhada
git commit -m "Título do commit" -m "Descrição detalhada..."

# Commit de todos os arquivos modificados (sem git add)
git commit -am "Mensagem do commit"

# Alterar último commit
git commit --amend -m "Nova mensagem"
```

### Boas Práticas para Mensagens de Commit

```
❌ Ruim: "arrumei bug"
❌ Ruim: "atualização"
❌ Ruim: "teste"

✅ Bom: "Corrige cálculo de imposto no módulo financeiro"
✅ Bom: "Adiciona validação de CPF no formulário de cadastro"
✅ Bom: "Remove dependência obsoleta do package.json"
```

**Estrutura ideal:**
```
<verbo no presente> + <o quê> + <porquê (opcional)>

Ex: "Adiciona autenticação JWT para segurança da API"
```

### Visualizando o Histórico

```bash
# Histórico completo
git log

# Histórico resumido (uma linha por commit)
git log --oneline

# Histórico com gráfico de branches
git log --oneline --graph --all

# Últimos 5 commits
git log -n 5

# Histórico de um arquivo específico
git log -- arquivo.txt

# Histórico com autor e data
git log --pretty=format:"%h - %an, %ar : %s"
```

### Trabalhando com Remotos

```bash
# Ver repositórios remotos
git remote -v

# Adicionar repositório remoto
git remote add origin https://github.com/usuario/repositorio.git

# Baixar alterações do remoto (sem mesclar)
git pull origin main

# Baixar e mesclar alterações
git pull

# Enviar alterações para o remoto
git push origin main

# Enviar pela primeira vez
git push -u origin main
```

### Visualizando Diferenças

```bash
# Ver diferenças entre working directory e staging
git diff

# Ver diferenças entre staging e último commit
git diff --staged

# Ver diferenças entre dois commits
git diff abc123 def456

# Ver diferenças de um arquivo específico
git diff arquivo.txt
```

## 2.6 Exemplo Prático Completo

Vamos criar um projeto do zero:

```bash
# 1. Crie uma pasta para o projeto
mkdir meu-projeto
cd meu-projeto

# 2. Inicialize o repositório Git
git init

# 3. Crie um arquivo README
echo "# Meu Projeto" > README.md
echo "Descrição do projeto" >> README.md

# 4. Verifique o status
git status
# Saída: Untracked files: README.md

# 5. Adicione o arquivo ao staging
git add README.md

# 6. Crie o commit
git commit -m "Adiciona README inicial"

# 7. Crie um repositório no GitHub e adicione como remote
git remote add origin https://github.com/seu-usuario/meu-projeto.git

# 8. Envie para o GitHub
git push -u origin main
```

## 2.7 Comandos Úteis do Dia a Dia

| Comando | Descrição |
|---------|-----------|
| `git status` | Mostra o estado dos arquivos |
| `git log --oneline` | Histórico resumido |
| `git diff` | Mostra alterações não staged |
| `git checkout -- arquivo` | Descarta alterações de um arquivo |
| `git reset HEAD arquivo` | Remove arquivo do staging |
| `git clean -fd` | Remove arquivos untracked |
| `git stash` | Guarda alterações temporariamente |
| `git branch -a` | Lista todas as branches |

## 2.8 Atalhos e Dicas

### Atalhos Comuns

```bash
# git st = git status
# git co = git checkout
# git br = git branch

# Adicione ao ~/.gitconfig:
[alias]
    st = status
    co = checkout
    br = branch
    ci = commit
    lg = log --oneline --graph --all
```

### Dicas Importantes

1. **Commite com frequência** - Commits pequenos são mais fáceis de revisar e reverter
2. **Escreva mensagens claras** - Daqui a 6 meses, você agradecerá
3. **Use branches** - Nunca trabalhe direto na `main`
4. **Pull antes do push** - Sempre atualize antes de enviar
5. **Revise antes de commitar** - `git diff --staged`
