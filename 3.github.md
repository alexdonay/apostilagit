# 3. GitHub

## 3.1 O que é o GitHub?

O **GitHub** é uma plataforma de hospedagem de repositórios Git baseada na nuvem. Pertence à Microsoft desde 2018 e é a maior plataforma de desenvolvimento colaborativo do mundo, com mais de 100 milhões de desenvolvedores.

### Principais Funcionalidades:

| Funcionalidade | Descrição |
|---------------|-----------|
| **Hospedagem de repositórios** | Armazene seus projetos Git na nuvem |
| **Colaboração** | Trabalhe em equipe com ferramentas integradas |
| **Code Review** | Revise código com Pull Requests |
| **Issues** | Gerencie bugs e tarefas |
| **Actions** | Automatize workflows (CI/CD) |
| **Pages** | Hospede sites estáticos gratuitamente |
| **Packages** | Gerencie pacotes e dependências |

### Alternativas ao GitHub:

- **GitLab** - Plataforma completa com CI/CD integrado
- **Bitbucket** - Integrado com ferramentas Atlassian (Jira, Trello)
- **Azure DevOps** - Solução Microsoft para empresas

## 3.2 Criando uma Conta

### Passo a Passo:

1. Acesse [https://github.com](https://github.com)
2. Clique em **"Sign up"** (Cadastrar)
3. Preencha as informações:
   - E-mail
   - Senha (mínimo 7 caracteres)
   - Nome de usuário (será seu @ no GitHub)
4. Verifique seu e-mail
5. Complete o questionário inicial (opcional)

### Tipos de Conta:

| Tipo | Preço | Recursos |
|------|-------|----------|
| **Free** | Gratuito | Repositórios públicos e privados ilimitados |
| **Pro** | $4/mês | Recursos avançados para indivíduos |
| **Team** | $4/usuário/mês | Ferramentas para equipes |
| **Enterprise** | $21/usuário/mês | Controle e segurança corporativa |

> 💡 **Dica:** A conta **Free** é mais que suficiente para aprender e para a maioria dos projetos pessoais!

## 3.3 Autenticação

### Métodos de Autenticação:

#### 1. HTTPS com Token de Acesso Pessoal (Recomendado)

O GitHub não aceita mais senha para operações Git via HTTPS. Use **Personal Access Tokens**:

**Criando um Token:**

1. Acesse **Settings** → **Developer settings** → **Personal access tokens** → **Tokens (classic)**
2. Clique em **"Generate new token (classic)"**
3. Dê um nome ao token (ex: "Meu Notebook")
4. Selecione as permissões (scopes):
   - ✅ `repo` (Full control of private repositories)
5. Clique em **"Generate token"**
6. **Copie o token imediatamente** (você não poderá vê-lo novamente!)

**Usando o Token:**

```bash
# Ao fazer push/pull, use o token no lugar da senha
git clone https://github.com/usuario/repositorio.git
# Quando pedir senha, cole o token

# Ou inclua o token na URL (não recomendado):
git clone https://SEU_TOKEN@github.com/usuario/repositorio.git
```

#### 2. SSH (Recomendado para Uso Frequente)

**Gerando uma Chave SSH:**

```bash
# Gerar nova chave SSH
ssh-keygen -t ed25519 -C "seu.email@exemplo.com"

# Ou usando RSA (para sistemas mais antigos)
ssh-keygen -t rsa -b 4096 -C "seu.email@exemplo.com"
```

**Adicionando a Chave ao GitHub:**

1. Copie o conteúdo da chave pública:
   ```bash
   cat ~/.ssh/id_ed25519.pub
   ```

2. No GitHub, acesse **Settings** → **SSH and GPG keys**

3. Clique em **"New SSH key"**

4. Cole o conteúdo da chave e dê um nome (ex: "Notebook Pessoal")

5. Clique em **"Add SSH key"**

**Testando a Conexão:**

```bash
ssh -T git@github.com
# Saída esperada: Hi usuario! You've successfully authenticated...
```

**Clonando via SSH:**

```bash
git clone git@github.com:usuario/repositorio.git
```

#### 3. Autenticação de Dois Fatores (2FA)

**Ativando 2FA:**

1. Acesse **Settings** → **Password and authentication**
2. Em "Two-factor authentication", clique em **"Enable 2FA"**
3. Escolha o método:
   - **Authenticator app** (Recomendado - Google Authenticator, Authy)
   - **SMS**
4. Siga as instruções para completar a configuração
5. **Salve os códigos de recuperação** em local seguro!

> ⚠️ **Importante:** Com 2FA ativado, você **precisa** usar Token ou SSH para operações Git!

## 3.4 Criando um Repositório

### Passo a Passo:

1. Acesse [https://github.com/new](https://github.com/new)
2. Preencha as informações:

| Campo | Descrição |
|-------|-----------|
| **Owner** | Sua conta ou organização |
| **Repository name** | Nome do repositório (ex: `meu-projeto`) |
| **Description** | Descrição opcional do projeto |
| **Public/Private** | Público (visível) ou Privado (só você vê) |
| **Initialize with README** | Cria um arquivo README.md inicial |
| **Add .gitignore** | Ignora arquivos desnecessários (recomendado) |
| **Add license** | Licença de uso do projeto |

3. Clique em **"Create repository"**

### Tipos de Visibilidade:

| Tipo | Quem pode ver | Quem pode contribuir |
|------|---------------|---------------------|
| **Público** | Qualquer pessoa | Apenas colaboradores convidados |
| **Privado** | Apenas você e colaboradores | Apenas colaboradores convidados |

### Arquivos Importantes:

| Arquivo | Finalidade |
|---------|-----------|
| `README.md` | Apresentação do projeto |
| `.gitignore` | Arquivos a serem ignorados pelo Git |
| `LICENSE` | Licença de uso do projeto |

**Exemplo de .gitignore para Python:**
```
__pycache__/
*.py[cod]
*.env
venv/
.DS_Store
```

## 3.5 Colaboração com Outros Desenvolvedores

### Adicionando Colaboradores (Repositórios Privados)

1. Acesse o repositório
2. Clique em **Settings** → **Collaborators**
3. Clique em **"Add people"**
4. Digite o username ou e-mail do colaborador
5. Selecione a pessoa e clique em **"Add collaborator"**

### Permissões em Organizações:

| Permissão | Descrição |
|-----------|-----------|
| **Read** | Apenas visualização |
| **Triage** | Gerencia issues e PRs |
| **Write** | Pode pushar no repositório |
| **Maintain** | Gerencia repositório (sem excluir) |
| **Admin** | Controle total |

## 3.6 Pull Requests (PR)

O **Pull Request** é a principal ferramenta de colaboração no GitHub. Ele permite que você proponha mudanças e peça revisão de código.

### Fluxo de um Pull Request:

```
1. Fork do repositório (se for projeto de terceiros)
        ↓
2. Cria uma branch para sua feature
        ↓
3. Faz as alterações e commita
        ↓
4. Push para o GitHub
        ↓
5. Abre o Pull Request
        ↓
6. Revisão da equipe (Code Review)
        ↓
7. Aprovação e Merge
```

### Criando um Pull Request:

1. Crie uma branch com sua feature:
   ```bash
   git checkout -b feature/nova-funcionalidade
   ```

2. Faça as alterações e commit:
   ```bash
   git add .
   git commit -m "Adiciona nova funcionalidade"
   git push origin feature/nova-funcionalidade
   ```

3. No GitHub, clique em **"Pull requests"** → **"New pull request"**

4. Selecione:
   - **Base repository:** `main` (ou branch principal)
   - **Head repository:** `feature/nova-funcionalidade`

5. Preencha:
   - Título descritivo
   - Descrição das mudanças
   - Link para issues relacionadas

6. Clique em **"Create pull request"**

### Revisando um Pull Request:

- **Files changed** - Visualiza as alterações
- **Comment** - Adiciona comentários gerais
- **Review changes** - Aprova, solicita mudanças ou comenta
- **Inline comments** - Comenta em linhas específicas

## 3.7 Issues (Gerenciamento de Tarefas)

**Issues** são usadas para rastrear bugs, tarefas, melhorias e discussões.

### Criando uma Issue:

1. Acesse a aba **Issues** do repositório
2. Clique em **"New issue"**
3. Preencha:
   - Título descritivo
   - Descrição detalhada
   - Labels (etiquetas)
   - Assignees (responsáveis)
   - Projects (quadros Kanban)

### Labels Comuns:

| Label | Descrição |
|-------|-----------|
| `bug` | Algo está quebrado |
| `enhancement` | Nova funcionalidade |
| `documentation` | Melhorias na documentação |
| `good first issue` | Boa para iniciantes |
| `help wanted` | Precisa de ajuda |

### Templates de Issue:

Muitos projetos usam templates padronizados:

```markdown
## Descrição
Descreva o problema ou solicitação.

## Passos para Reproduzir (bug)
1. Passo 1
2. Passo 2
3. Erro ocorre

## Comportamento Esperado
O que deveria acontecer?
```

## 3.8 GitHub Actions (CI/CD)

**GitHub Actions** automatiza workflows diretamente no GitHub.

### Exemplo de Workflow:

```yaml
# .github/workflows/ci.yml
name: CI

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v4
    
    - name: Setup Node.js
      uses: actions/setup-node@v4
      with:
        node-version: '18'
    
    - name: Install dependencies
      run: npm install
    
    - name: Run tests
      run: npm test
```

### Casos de Uso:

- ✅ Executar testes automaticamente
- ✅ Deploy automático em produção
- ✅ Validação de código (linting)
- ✅ Geração de documentação
- ✅ Notificações em Slack/Discord

## 3.9 GitHub Pages

Hospede sites estáticos gratuitamente!

### Configurando:

1. Acesse **Settings** → **Pages**
2. Em **Source**, selecione a branch (ex: `main`)
3. Selecione a pasta (ex: `/root` ou `/docs`)
4. Clique em **Save**

Seu site estará disponível em:
```
https://seu-usuario.github.io/nome-do-repositorio
```
